## Milestone Summary

**Key Decisions:**

- 1ms threshold guideline for GIL release decisions
- YAML operations have architectural limitation - Python<->Rust dict conversion requires GIL
- Quick/thorough benchmark modes via BENCH_MODE env var (quick: 50 samples, thorough: 200 samples)
- Shared benchmark config via #[path] attribute (not crate dependency)
- Real crash logs embedded via include_str! for scanlog benchmarks
- Native frames enabled by default in py-spy for combined Python+Rust stacks
- AtomicU64 with Ordering::Relaxed for cache hit/miss counters (lock-free statistics)
- serial_test crate for serializing Rust tests that touch global state
- ResourceLoader.get_data_directory().parent as project root anchor
- cProfile used instead of py-spy due to Python 3.14 incompatibility
- Threading overhead identified as dominant factor (86%) for optimization
- Python algorithmic improvements (O(n) to O(1)) more impactful than Rust micro-optimizations
- ready_for_review trigger for benchmark CI (reduces noise vs all PR events)
- 5% warning / 10% failure thresholds as CI defaults
- Label bypass (perf-regression-accepted) for intentional regressions

**Issues Resolved:**

- BUG-01: test_clear_cache parallel test pollution (fixed with #[serial])
- BUG-02: classic_settings() CWD-dependent path resolution (fixed with ResourceLoader anchor)
- GIL benchmarks not using shared config (retrofitted in Phase 18)
- dump_cache_stats.ps1 using deprecated API (updated in Phase 18)
- Missing profiling workflow documentation (added in Phase 18)

**Issues Deferred:**

- py-spy 0.4.1 incompatible with Python 3.14 (external dependency, requires py-spy update)
- Human verification of CI workflow runtime behavior (requires actual PR trigger)

**Technical Debt Incurred:**

- None significant - Phase 18 addressed all audit-identified tech debt
