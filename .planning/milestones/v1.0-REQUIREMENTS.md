## v1 Requirements

Requirements for the cleanup milestone. Each maps to roadmap phases.

### Dead Code Removal

- [x] **DEAD-01**: Remove all files with DEPRECATED markers (11 files identified across ClassicLib/) — *Validated: removed deprecated modules including _DeprecatedVersion, database_rust.py shim*
- [x] **DEAD-02**: Add vulture dead code detection to CI pipeline to prevent regression — *Validated: Vulture 2.14 installed, whitelist curated, CI enforcement active*
- [x] **DEAD-03**: Audit all 39 Rust crates (21 business-logic + 18 python-bindings) and remove stub/empty crates — *Validated: all crates audited, stubs removed*
- [x] **DEAD-04**: Establish test coverage baseline with pytest --cov and identify 0% coverage modules as deletion candidates — *Validated: 71% coverage baseline established*

### Global State Cleanup

- [x] **GLOB-01**: Replace mutable global flags (_VERSION_WARNING_LOGGED and similar) with instance variables or functools.lru_cache with cache_clear() — *Validated: lru_cache(maxsize=1) pattern adopted*
- [x] **GLOB-02**: Audit and categorize all 19 `global _*` instances: lazy-init caches (keep), mutable flags (replace with instance state), test-hostile (add reset methods) — *Validated: all 19 categorized and addressed*
- [x] **GLOB-03**: Add reset_all_singletons() autouse fixture for comprehensive test isolation across all singleton state — *Validated: fixture covers 19+ globals in 4 categories*

### Redundancy Removal

- [x] **REDN-01**: Remove FormIDAnalyzer sync wrapper (156 lines wrapping FormIDAnalyzerCore with deprecated create_sync_wrapper) — *Validated: FormIDAnalyzer.py deleted, callers use FormIDAnalyzerCore directly*
- [x] **REDN-02**: Consolidate YAML sync/async split — merge parallel sync/ and async_/ cache implementations into async-only with sync wrappers at GUI entry points only — *Validated: yaml/sync/ directory eliminated, modules merged to yaml/ level*
- [x] **REDN-03**: Remove Python fallback implementations (8 files in ClassicLib/integration/python/) after verifying Rust wheel bundling and test coverage equivalence — *Validated: all 8 fallbacks removed, integration/python/ directory eliminated entirely*
- [x] **REDN-04**: Remove all create_sync_wrapper usage and transitional bridge helpers (_async_utils/bridge_helpers.py, io/files/sync_adapters.py) — *Validated: 396 lines of sync infrastructure deleted, 8 callers migrated*

### Architecture Simplification

- [x] **ARCH-01**: Simplify factory/detector/status three-layer indirection — collapse to single-layer factory with try-import pattern, remove duplicate caching between _components_cache and _detection_cache — *Validated: single flat factory.py (~948 lines), 80+ files updated*
- [x] **ARCH-02**: Remove or simplify RustAcceleration coordinator package (ClassicLib/acceleration/ — coordinator.py, metrics.py, types.py, workload.py, ~400 lines) after auditing runtime usage — *Validated: 960-line acceleration/ package deleted entirely*
- [x] **ARCH-03**: Type-narrow all factory function returns from Any to specific Rust types (~8 factory functions), enabling Pyright strict mode to catch errors — *Validated: 13 Protocol types created, pyright passes with 0 errors*
- [x] **ARCH-04**: Thin out fat Python wrappers by moving business logic from ClassicLib/integration/rust/ wrappers into Rust -core crates (file_io_rust.py 39KB, parser 15KB, formid 16KB) — *Validated: 60-75% reduction achieved (accepted deviation: file_io at 230 lines vs 200 target)*
