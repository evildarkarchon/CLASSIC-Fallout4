<objective>
Bound the process-global hash cache and give it the same observability contract as YAML and settings so all three caches can be surfaced consistently through bindings.

Purpose: CACHE-03 currently has only `cache_size()` and an unbounded DashMap-backed cache. This plan introduces the 1024-entry bounded store and the canonical Phase 4 stats contract while preserving explicit clear behavior per D-01, D-02, D-06, and D-07. It also follows the research-backed reconciliation that roadmap `LRU` language must be implemented as bounded `quick_cache` eviction, not strict victim-order verification.
Output: A bounded hash cache, public hash cache stats/reset helpers, updated tests, and synced file-io docs.
</objective>
