The root cause is that ScanLogsExecutor.__init__ (line 83) calls crashlogs_get_files(), which calls classic_settings() and yaml_settings(). These sync functions detect async context and raise RuntimeError.

Fix approach: Move ScanLogsExecutor creation to BEFORE asyncio.run(), so it runs in sync context.

1. In CLASSIC_ScanLogs.py:
   - Move `executor = ScanLogsExecutor(config)` creation to main() function, BEFORE asyncio.run()
   - Pass the executor instance to run_scan() instead of args
   - This requires creating config synchronously before asyncio.run()

2. Create a sync config creation function in CLASSIC_ScanLogs.py:
   - Extract the non-async portions of create_config_from_args_async()
   - YAML writes for persisting args can remain async (they happen during execution)
   - The key is that ScanLogsExecutor creation happens in sync context

3. Update run_scan() signature:
   - Accept executor instead of args
   - Handle any remaining async config updates during scan

Key constraint: Do NOT modify yaml_settings() or classic_settings() - they correctly enforce sync-only usage. Fix the caller.

Reference the existing code patterns:
- SetupCoordinator.initialize_application() already runs in sync context before asyncio.run()
- GUI mode creates ScanLogsExecutor in Qt sync callbacks, which works fine
