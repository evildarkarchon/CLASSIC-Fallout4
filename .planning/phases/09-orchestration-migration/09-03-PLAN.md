  <action>
The root cause is that ScanLogsExecutor.__init__ (line 83) calls crashlogs_get_files(), which calls classic_settings() and yaml_settings(). These sync functions detect async context and raise RuntimeError when __init__ runs inside asyncio.run().

**Implementation approach: Option A - Lazy initialization**

This mirrors the existing lazy pattern used for `yamldata` (lines 92-94) which defers loading until execute_scan().

1. In ScanLogsExecutor.__init__() (line 69+):
   - REMOVE line 83: `self.crashlog_list: list[Path] = crashlogs_get_files()`
   - REPLACE with: `self.crashlog_list: list[Path] = []` (empty placeholder)
   - REMOVE lines 103-104 that set statistics.total_files (defer to execute_scan)
   - Keep the comment about lazy initialization

2. Add new async method `_ensure_crashlog_list_async()`:
   ```python
   async def _ensure_crashlog_list_async(self) -> None:
       """Lazily load crash log files using async-safe method.

       This method is called during execute_scan() to avoid sync settings
       calls from async context. Uses asyncio.to_thread() to run the sync
       crashlogs_get_files() in a thread pool, avoiding async context detection.
       """
       if self.crashlog_list:
           return  # Already loaded

       # Run sync function in thread pool to avoid async context detection
       self.crashlog_list = await asyncio.to_thread(crashlogs_get_files)
       self.statistics.total_files = len(self.crashlog_list)
       logger.debug(f"Lazily loaded {len(self.crashlog_list)} crash log files")
   ```

3. In _initialize_scan_resources() (line 212+):
   - Add call to `await self._ensure_crashlog_list_async()` FIRST, before yamldata loading
   - This ensures crash logs are loaded before any processing begins

4. Update scan_sync() (line 382+) for GUI compatibility:
   - The existing GUI flow uses scan_sync() which calls execute_scan() via AsyncBridge
   - AsyncBridge creates its own event loop, so the thread pool approach will work

**Why this approach:**
- Mirrors existing yamldata lazy pattern (already proven to work)
- No changes needed to CLASSIC_ScanLogs.py
- No changes needed to crashlogs_get_files() or yaml_settings()
- asyncio.to_thread() runs the sync function in a thread pool, which does NOT have an event loop running, so async context detection returns False
- GUI compatibility preserved (AsyncBridge creates isolated event loop)

**Key constraint:** Do NOT modify yaml_settings() or classic_settings() - they correctly enforce sync-only usage. Fix the caller.
