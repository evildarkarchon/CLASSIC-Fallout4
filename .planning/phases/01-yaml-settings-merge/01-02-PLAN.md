<objective>
Consolidate all three binding layers (C++ bridge, Node, Python) to use settings-core instead of yaml-core, update C++ consumers, AND expand the C++ bridge to cover **all FFI-safe classic-settings-core surface plus documented exceptions** (cache ops + validators) per D-09.

Purpose: After Plan 01 merged the Rust core, the binding layers still have separate yaml modules/crates. This plan merges them per D-05 through D-09: renames the C++ bridge module and updates all C++ consumers, adds NEW bridge functions for settings-core cache ops and validators (closing the parity gap with Python/Node — D-09), merges Node yaml.rs into settings.rs, folds Python yaml-py into settings-py, and deletes the yaml-py crate. Binding tests are also merged/updated.

**Reviews-mode concern 12 note:** The C++ bridge is not literally the "full" settings-core surface. It covers every function that can cross the CXX boundary given the CXX type system, with two explicit documented exceptions: (a) `get_cached` cannot return `Option<Arc<Vec<Yaml>>>` across CXX and is skipped; (b) the `load_*` family returns a document-count `u32` rather than the full `Vec<Yaml>` payload because raw `Yaml` cannot cross CXX either. C++ callers that need the parsed docs must go through the existing `yaml_ops_*` load helpers. Everything else on the settings-core cache and validator surface IS exposed.

Output: Three unified binding surfaces with no yaml-specific modules or crates remaining, and a C++ bridge that matches the Python settings-core surface subject to the two documented CXX-type exceptions above.
</objective>
