1. **`_parse_crash_header` static method** -- Already exists in Rust's `LogParser.parse_crash_header()`. Remove entirely from Python.

5. **Exception tuple construction** -- Replace with direct exception imports.
6. **try/except/fallback chains** -- Simplify to `if self._rust_parser: ... else: fallback`.

**Step 4: Thin parser_rust.py.**

Rewrite following thin delegation pattern:

```python
def find_segments(self, crash_data: list[str], crashgen_name: str,
                  xse_acronym: str, game_root_name: str) -> tuple[str, str, str, dict]:
    """Find crash log segments."""
    if self._rust_parser:
        return self._rust_parser.find_segments(crash_data, crashgen_name, xse_acronym, game_root_name)
    return self._python_fallback.find_segments(crash_data, crashgen_name, xse_acronym, game_root_name)
```

Rules:
- Keep class docstring (shortened to ~5 lines)
- Keep `__init__` with Rust detection and Python fallback setup
- Remove `_parse_crash_header` (already in Rust)
- Remove `hasattr` feature detection
- Remove legacy multi-call FFI path
- Each method: 3-5 lines (docstring + if/else delegation)

**Step 3: Thin formid_rust.py.**

Rewrite following thin delegation pattern. For `formid_match`, if it creates ReportFragment objects from Rust results, that IS valid wrapper work (type conversion). Keep it but minimize:

```python
def formid_match(self, formids: list[str], plugins: list[str],
                 report: ReportFragment) -> None:
    """Match FormIDs against database."""
    if self._rust_analyzer:
        result = self._rust_analyzer.formid_match(formids, plugins)
        # Type conversion: Rust result -> ReportFragment lines
        for line in result.lines:
            report.add_line(line)
        return
    self._python_fallback.formid_match(formids, plugins, report)
```

Rules:
- Same thin delegation pattern as parser
- ReportFragment creation from results is acceptable wrapper work
- Remove ALL caching from Python (let Rust handle or accept no caching for now)
- Remove multi-path init, hasattr, exception tuples
- Each method: 3-8 lines (allowing slightly more for type conversion)
