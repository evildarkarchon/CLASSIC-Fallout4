**Step 3: Identify what to eliminate vs. keep.**

From the research, the wrapper fat comes from:
1. **SyncWrapper inner class** -- Full API duplication for sync callers. Check if ANY production code (not tests) calls `create_file_io_sync()`. If no production callers, remove entirely. If production callers exist, convert them to use FileIOCore directly.
2. **`get_rust_file_io()` convenience function** -- Check callers, remove if unused outside tests.
3. **Check if ANY production code (not tests) calls `create_file_io_sync()`:**
   ```bash
   grep -r "create_file_io_sync\|SyncWrapper" ClassicLib/ --include="*.py" | grep -v test
   ```
   If no production callers, remove entirely. If production callers exist, convert them to use FileIOCore directly.
4. **`_ensure_path()` helper** -- Rust PathLike handles this; remove.
5. **`_get_rust_exception_types()` and exception tuple construction** -- Replace with direct exception imports from `ClassicLib.integration.exceptions`.
5. **Per-method try/except/fallback chains** -- Keep the `if self._rust_core:` / `else: fallback` structure but remove verbose exception chaining. Each method body should be: convert args -> call Rust -> convert return. Fallback path stays for Phase 5.
6. **`read_crash_log` trailing line strip logic** -- If this logic exists only in Python, note it for migration to Rust in the NEXT task. For now, keep it as a thin helper if needed.
7. **`write_crash_report` extension conversion** -- Same approach as above.

**Step 4: Thin the wrapper.**

Rewrite `file_io_rust.py` following this pattern for EVERY method:

```python
async def read_file(self, path: Path, encoding: str = "utf-8") -> str:
    """Read file contents."""
    if self._rust_core:
        return await self._rust_core.read_file(str(path), encoding)
    return await self._python_fallback.read_file(path, encoding)
```

For blocking methods:
```python
def read_dds_header(self, path: Path) -> dict[str, Any]:
    """Read DDS header."""
    if self._rust_core:
        return self._rust_core.read_dds_header(str(path))
    return self._python_fallback.read_dds_header(path)
```

Rules:
- Keep the class docstring (shortened to ~10 lines max)
- Keep `__init__` with Rust detection and Python fallback setup
- Remove ALL inner classes (SyncWrapper)
- Remove ALL module-level convenience functions (create_file_io_sync, get_rust_file_io)
- Remove `_ensure_path`, `_get_rust_exception_types`
- Each method: 3-5 lines max (docstring + if/else delegation)
- Keep `RUST_AVAILABLE` module-level flag (used by tests and detection)
- If `read_crash_log` or `write_crash_report` have business logic (trailing strip, extension conversion), keep a MINIMAL helper (1-2 lines) or inline it -- do NOT add complex logic

**Step 5: Update __init__.py exports.**
