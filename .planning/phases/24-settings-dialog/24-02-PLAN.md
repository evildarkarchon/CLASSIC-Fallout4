**Create `settings.rs`:**

Module doc: "Settings persistence and validation for the GUI. Provides load, save, validate, and reset functions using ClassicConfig and YAML persistence."

Key design principles per CONTEXT.md and RESEARCH.md:
- Live save-on-change: Each setting change writes to YAML immediately
- Initialization guard: All save functions take an `initialized: bool` parameter and return early if false
- Path validation: Only save paths to directories that exist
- Settings file location: Use `directories::ProjectDirs::from("com", "classic", "classic-gui")` config_dir, same directory as window_state.json

**Functions to implement:**

1. `pub fn settings_file_path() -> Option<PathBuf>` -- returns the path to `settings.yaml` in the ProjectDirs config directory. Use same pattern as `state.rs`'s `state_file_path()`.

2. `pub fn load_settings() -> ClassicConfig` -- loads settings from the settings file path. Uses `classic_shared_core::get_runtime().block_on()` to call `ClassicConfig::load_from_yaml()`. Falls back to `ClassicConfig::default()` if file doesn't exist or fails to parse. Handles VR Mode migration (this happens automatically in ClassicConfig::from_yaml per Task 1 of Plan 01).

3. `pub fn save_full_config(config: &ClassicConfig) -> Result<(), Box<dyn std::error::Error>>` -- saves the entire ClassicConfig to YAML. Uses `get_runtime().block_on(config.save_to_yaml(path))`. Creates parent directory if needed.

4. `pub fn save_setting_bool(config: &mut ClassicConfig, key: &str, value: bool) -> Result<(), Box<dyn std::error::Error>>` -- updates the named field in config and saves the full config. Match key string to config field:
   - "fcx_mode" => config.fcx_mode = value
   - "update_check" => config.update_check = value
   - "simplify_logs" => config.simplify_logs = value
   - "show_formid_values" => config.show_formid_values = value
   - "move_unsolved_logs" => config.move_unsolved_logs = value
   - "auto_switch_to_results" => config.auto_switch_to_results = value
   - _ => return Err("Unknown bool setting")
   Then call `save_full_config(config)`.

5. `pub fn save_setting_string(config: &mut ClassicConfig, key: &str, value: &str) -> Result<(), Box<dyn std::error::Error>>` -- updates a string field and saves:
   - "game_version" => config.game_version = value.to_string()
   - "update_source" => config.update_source = value.to_string()
   - _ => return Err("Unknown string setting")
   Then call `save_full_config(config)`.

6. `pub fn save_path_setting(config: &mut ClassicConfig, key: &str, path: &str) -> Result<(), String>` -- validates path exists as directory, then updates config and saves:
   - Validate: if `path.trim().is_empty()`, clear the path (set to None) and save (empty = cleared).
   - Validate: if not empty, check `Path::new(path).is_dir()`. If not a directory, return `Err(format!("Directory does not exist: {}", path))`.
   - Match key to config field:
     - "ini_folder" => config.paths.ini_folder = Some/None
     - "mods_folder" => config.paths.mods_folder = Some/None
     - "scan_custom" => config.paths.scan_custom = Some/None
   - Call `save_full_config(config).map_err(|e| e.to_string())`

7. `pub fn reset_to_defaults() -> ClassicConfig` -- creates `ClassicConfig::default()`, saves it via `save_full_config()`, and returns the defaults. If save fails, log the error but still return defaults.

8. `pub fn detect_game_version(config: &ClassicConfig) -> String` -- basic game version detection stub:
   - If `config.paths.game_root` is empty or doesn't exist, return `"(detection unavailable)"`.
   - Check for `Fallout4VR.exe` in game_root. If found, return `"(detected: VR)"`.
   - Otherwise return `"(detected: NextGen)"` as a reasonable default for Fallout 4.
   - This is a stub per RESEARCH.md -- full detection can be enhanced later.

9. `pub fn game_version_index_to_string(index: i32) -> &'static str` -- maps ComboBox index to storage string: 0 => "auto", 1 => "Original", 2 => "NextGen", 3 => "VR".

10. `pub fn game_version_string_to_index(version: &str) -> i32` -- maps storage string to ComboBox index. Case-insensitive matching. Default to 0 (auto) for unrecognized values.

11. `pub fn update_source_index_to_string(index: i32) -> &'static str` -- 0 => "github", 1 => "both".

12. `pub fn update_source_string_to_index(source: &str) -> i32` -- case-insensitive, default 0.

**Important:** Do NOT use `get_runtime().block_on()` inside an async context (would panic). The GUI callbacks execute in the Slint event loop (sync context), so `block_on()` is safe there. But the save is I/O -- for a snappier UX, keep saves synchronous since ClassicConfig::save_to_yaml uses spawn_blocking internally. If `block_on` causes issues because the runtime is already entered, use `std::thread::spawn` with a channel or simply write the YAML synchronously using `std::fs::write` directly instead of the async path. The simplest approach: build the YAML string via `config.to_yaml()` + `YamlEmitter`, then `std::fs::write()`. However, `to_yaml()` and `from_yaml()` are private on ClassicConfig. So either:
- Option A: Make `to_yaml()` pub on ClassicConfig (requires modifying classic-config-core)
- Option B: Use `save_to_yaml`/`load_from_yaml` with `tokio::runtime::Handle::current().block_on()` (we're NOT in an async context in callbacks, so `get_runtime().block_on()` is fine)
- Option C: Use `std::thread::spawn` to run the async save on a background thread

**Recommended: Option B** -- use `get_runtime().block_on(config.save_to_yaml(&path))` since GUI callbacks run in the Slint event loop thread which is NOT a Tokio async context. This is the same pattern used elsewhere in the GUI (e.g., `load_window_state` is sync).

For browse callbacks:
```rust
window.on_setting_ini_browse(move || {
    let window_weak = window_weak.clone();
    let state = Arc::clone(&state);
    let current = window_weak.upgrade()
        .map(|w| w.get_setting_ini_path().to_string())
        .unwrap_or_default();
    let start_dir = if current.is_empty() { None } else { Some(current) };

    AsyncBridge::run_with_ui_update(
        async move { browse_folder("Select INI Folder", start_dir.as_deref()).await },
        move |result| {
            if let Some(path) = result {
                let mut s = state.lock();
                if !s.initialized { return; }
                match save_path_setting(&mut s.settings, "ini_folder", &path) {
                    Ok(()) => {
                        if let Some(w) = window_weak.upgrade() {
                            w.set_setting_ini_path(path.into());
                            w.set_setting_ini_has_error(false);
                            w.set_setting_ini_error("".into());
                        }
                    }
                    Err(e) => {
                        if let Some(w) = window_weak.upgrade() {
                            w.set_setting_ini_has_error(true);
                            w.set_setting_ini_error(e.into());
                        }
                    }
                }
            }
        },
    );
});
```
