```python
    GlobalRegistry.get_game(),
    GlobalRegistry.is_vr_version(),
)
```

4. **Simplify `find_game_path()` (sync version):**
   - Check cached path from ResourceLoader.get_cached_game_path()
   - If cached and valid (use Rust PathValidator.is_valid_path), register and return
   - Get XSE log path from YAML settings
   - Call Rust `self._rust_finder.find_game_path(cached_path=..., xse_log_path=...)`

   - Same logic but use `asyncio.get_running_loop().run_in_executor()` for Rust calls
   - Use yaml_settings_async for YAML loading
   - Use ResourceLoader.save_path_to_cache_async for saving
   - **MUST register detected path:** `GlobalRegistry.register(GlobalRegistry.Keys.GAME_PATH, game_path)`

6. **Remove these Python implementation methods entirely:**
   - `_parse_xse_log_for_path()` - Rust handles this via `GamePathFinder.parse_xse_log()`
   - `_validate_game_path()` - Use Rust `PathValidator.validate_required_files()`

7. **Keep these methods (they handle user interaction, not detection):**
   - `_validate_xse_file()` - validates XSE log exists before parsing
   - `_report_xse_error()` - user messaging
   - `_get_path_from_user_gui()` - GUI dialog
   - `_get_path_from_user_console()` - console input
   - `_save_game_path()` / `_save_game_path_async()` - saves to cache

8. **Update `_game_path_find_registry()` function:**
   - Remove Python winreg fallback entirely
   - Keep only Rust path:
```python
def _game_path_find_registry(exe_name: str) -> Path | None:
    """Find game path via registry using Rust GamePathFinder."""
    try:
        finder = RustGamePathFinder(
            exe_name, None, GlobalRegistry.get_game(), GlobalRegistry.is_vr_version()
        )
        path_str = finder.find_game_path(cached_path=None, xse_log_path=None)
        game_path = Path(path_str)
        ResourceLoader.save_path_to_cache(game_path, "GamePath")
        GlobalRegistry.register(GlobalRegistry.Keys.GAME_PATH, game_path)
        return game_path
    except FileNotFoundError:
        logger.debug("Registry lookup failed: game not found")
        return None
```

9. **Keep game_generate_paths() and game_generate_paths_async() unchanged** - these do YAML settings manipulation, not detection.
