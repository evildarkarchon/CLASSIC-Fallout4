## Objective

Complete the benchmark regression detection system by adding PR comment posting and fail-on-regression enforcement. This plan transforms the analysis results from Plan 17-02 into actionable feedback for developers.

## Context

Plan 17-02 added comparison logic, threshold analysis with config file support, and label bypass detection. This plan:
- Posts/updates PR comments with benchmark results
- Fails the build when regressions exceed 10%
- Respects bypass label for intentional regressions

**User decisions (LOCKED):**
- PR comment: table format, update existing comment
- Failure messages: actionable summary with % regression
- Required check: benchmark must pass before merge
- Label bypass: `perf-regression-accepted` allows merge

## Tasks

<task id="1">
Configure PR comment posting

Add peter-evans/find-comment and peter-evans/create-or-update-comment actions:

```yaml
- name: Find existing comment
  if: github.event_name == 'pull_request'
  uses: peter-evans/find-comment@v3
  id: fc
  with:
    issue-number: ${{ github.event.pull_request.number }}
    comment-author: 'github-actions[bot]'
    body-includes: '<!-- benchmark-results -->'

- name: Build PR comment body
  if: github.event_name == 'pull_request'
  id: comment-body
  working-directory: rust
  shell: bash
  run: |
    # Build the comment body
    {
      echo '<!-- benchmark-results -->'
      echo '## Benchmark Results'
      echo ''
      echo "**Commit:** ${{ github.event.pull_request.head.sha }}"
      echo "**Baseline:** main branch"
      echo "**Mode:** quick (50 samples)"
      echo ''

      if [ "${{ steps.baseline-check.outputs.baseline_exists }}" != "true" ]; then
        echo '> [!WARNING]'
        echo '> No baseline found. Comparison skipped.'
        echo '> Baseline will be established when this PR merges to main.'
      else
        # Status badge
        status="${{ steps.analyze.outputs.status }}"
        case "$status" in
          failure) echo '> [!CAUTION]' && echo '> Performance regressions detected (>10%)' ;;
          warning) echo '> [!WARNING]' && echo '> Performance warnings detected (5-10%)' ;;
          success) echo '> [!TIP]' && echo '> No significant performance changes detected' ;;
          *) echo '> [!NOTE]' && echo '> Comparison status: '"$status" ;;
        esac

        echo ''

        # Failures table
        if [ -s failures.md ]; then
          echo '### Regressions (>10%)'
          echo ''
          echo '| Benchmark | Change | Current | Baseline |'
          echo '|-----------|--------|---------|----------|'
          cat failures.md
          echo ''
        fi

        # Warnings table
        if [ -s warnings.md ]; then
          echo '### Warnings (5-10%)'
          echo ''
          echo '| Benchmark | Change | Current | Baseline |'
          echo '|-----------|--------|---------|----------|'
          cat warnings.md
          echo ''
        fi

        # Improvements table
        if [ -s improvements.md ]; then
          echo '### Improvements'
          echo ''
          echo '| Benchmark | Change | Current | Baseline |'
          echo '|-----------|--------|---------|----------|'
          cat improvements.md
          echo ''
        fi

        # No changes case
        if [ ! -s failures.md ] && [ ! -s warnings.md ] && [ ! -s improvements.md ]; then
          echo 'All benchmarks within threshold (no changes >5%).'
          echo ''
        fi
      fi

      echo '---'
      echo '*Benchmarks run on GitHub Actions. Results may vary due to shared infrastructure.*'
      echo '*To accept regressions, add label `perf-regression-accepted` with justification.*'
    } > comment.md

    # Store as output (escape newlines)
    body=$(cat comment.md)
    echo "body<<EOF" >> $GITHUB_OUTPUT
    echo "$body" >> $GITHUB_OUTPUT
    echo "EOF" >> $GITHUB_OUTPUT

- name: Post or update PR comment
  if: github.event_name == 'pull_request'
  uses: peter-evans/create-or-update-comment@v4
  with:
    comment-id: ${{ steps.fc.outputs.comment-id }}
    issue-number: ${{ github.event.pull_request.number }}
    body: ${{ steps.comment-body.outputs.body }}
    edit-mode: replace
```

The comment uses GitHub's alert syntax (> [!CAUTION], etc.) for visual clarity.
</task>

<task id="2">
Add fail-on-regression step

Add final step that fails the workflow when regressions exceed threshold and bypass label is not present:

```yaml
- name: Fail on regression
  if: |
    github.event_name == 'pull_request' &&
    steps.analyze.outputs.has_failure == 'true' &&
    steps.check-label.outputs.bypass != 'true'
  shell: bash
  run: |
    echo "::error::Performance regression detected (>10%)"
    echo ""
    echo "One or more benchmarks showed significant regression."
    echo "Options:"
    echo "  1. Fix the performance regression"
    echo "  2. Add label 'perf-regression-accepted' with justification comment"
    echo ""
    echo "See the PR comment for detailed benchmark results."
    exit 1
```

This step uses `if` conditions to only fail when:
- This is a PR (not main branch push)
- Analysis found failures (>10% regressions)
- Bypass label is NOT present
</task>

   - Expected: Check passes with notice

6. **Main branch update**: Merge PR to main
   - Expected: Baseline cache updated with new results
</task>
