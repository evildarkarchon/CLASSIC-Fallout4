<task type="auto">
  <name>Task 2: Implement baseline caching and existence check</name>
  <files>.github/workflows/benchmarks.yml</files>
  <action>
Add cache and baseline check steps to the workflow:

**Cache restore (PR builds - restore only):**
```yaml
- name: Restore baseline cache
  id: baseline-cache
  uses: actions/cache/restore@v4
  with:
    path: rust/target/criterion/baseline
    key: criterion-baseline-${{ runner.os }}-dummy  # Never matches, forces restore-keys
    restore-keys: criterion-baseline-${{ runner.os }}
```

**Baseline existence check:**
```yaml
- name: Check baseline exists
  id: baseline-check
  working-directory: rust
  shell: bash
  run: |
    if [ -d "target/criterion/baseline" ]; then
      echo "baseline_exists=true" >> $GITHUB_OUTPUT
      echo "Baseline found - comparison will be performed"
    else
      echo "baseline_exists=false" >> $GITHUB_OUTPUT
      echo "::warning::No baseline found - skipping comparison. Run benchmarks on main branch first."
    fi
```

**Cache save (main branch only):**
```yaml
- name: Copy current to baseline (main only)
  if: github.ref == 'refs/heads/main'
  working-directory: rust
  shell: bash
  run: |
    if [ -d "target/criterion/current" ]; then
      rm -rf target/criterion/baseline
      cp -r target/criterion/current target/criterion/baseline
    fi

- name: Save baseline cache (main only)
  if: github.ref == 'refs/heads/main'
  uses: actions/cache/save@v4
  with:
    path: rust/target/criterion/baseline
    key: criterion-baseline-${{ runner.os }}-${{ github.sha }}
```

Key design: PRs restore existing baseline but never save. Main branch runs always save a new baseline keyed by commit SHA.
