<task type="auto">
  <name>Task 1: Create coverage measurement scripts</name>
  <files>
    rust/coverage_report.ps1
    rust/coverage_summary.ps1
  </files>
  <action>
Create two PowerShell scripts in `rust/`:

**coverage_report.ps1** - Main coverage runner:
- Accepts optional `-Package` parameter for per-crate runs (e.g., `./coverage_report.ps1 -Package classic-scanlog-core`)
- Without `-Package`, runs workspace-wide coverage
- Uses two-phase approach: `cargo llvm-cov --no-report` first, then generates HTML + lcov + JSON
- Exclusion regex: `"(target[/\\\\]|build[/\\\\]|\.slint)"` for all runs (excludes Slint generated code, build artifacts)
- For workspace runs, explicitly pass `--features gui-bridge` for classic-shared-core's AsyncBridge tests (research Pitfall 6)
- Output directories:
  - HTML: `target/llvm-cov/html/`
  - JSON: `target/llvm-cov/coverage.json`
  - lcov: `target/llvm-cov/lcov.info`
- Print summary at end showing output paths and overall coverage percentage

**coverage_summary.ps1** - Per-crate gap analysis:
- Reads `target/llvm-cov/coverage.json` (must run coverage_report.ps1 first)
- Parses JSON to extract per-crate line coverage percentages
- Outputs a sorted table: crate name, lines covered, lines total, coverage %, status (PASS if >=60%, GAP if <60%)
- Highlights crates below 60% threshold
- Counts total crates at/above 60% vs below
- Note: JSON format from cargo-llvm-cov uses LLVM export format. The structure has `data[0].files[]` with `filename`, `summary.lines.count`, `summary.lines.covered`. Group files by crate path (e.g., files containing `business-logic/classic-scanlog-core/` belong to that crate).

IMPORTANT: Do NOT write to NUL or nul (Windows special file). If suppressing output is needed, use `Out-Null` or `$null =` in PowerShell.
  </action>
