**main.rs -- Migrate scan callback to use run_cancellable:**

The current scan callback (in `setup_scan_callbacks`, around line 333) does:
1. Creates a CancellationToken
2. Stores it in AppState
3. Calls `AsyncBridge::run_with_ui_update(scan_crash_logs(window_weak, cancel_token, path), |result| ...)`

Migrate to `run_cancellable`:
1. Keep CancellationToken creation and AppState storage (needed for cancel button)
2. Change the `scan_crash_logs` function call -- it currently takes the CancellationToken as a parameter for inner-loop cancellation checks (per-log `is_cancelled()`). The `run_cancellable` handles OUTER cancellation (racing the whole future). Both are needed:
   - `run_cancellable` catches cancellation between await points
   - `scan_crash_logs` checks `is_cancelled()` in the per-log loop for responsiveness

So the migration is:
```rust
AsyncBridge::run_cancellable(
    cancel_token.clone(),  // Bridge handles outer cancellation
    classic_gui::scan_crash_logs(window_weak.clone(), cancel_token, crash_log_path),
    move |result| {
        // result is Option<Result<ScanResult, String>>
        // None = cancelled at the bridge level
        // Some(Ok(scan_result)) = completed or cancelled at the inner level
        // Some(Err(msg)) = error
        if let Some(w) = window_weak_completion.upgrade() {
            match result {
                None => {
                    // Bridge-level cancellation
                    w.set_scan_progress(0.0);
                    w.set_scan_status("Cancelled".into());
                    w.set_scan_in_progress(false);
                }
                Some(Ok(scan_result)) => {
                    // ... same handling as current code ...
                }
                Some(Err(msg)) => {
                    w.set_scan_progress(0.0);
                    w.set_scan_status(msg.into());
                    w.set_scan_in_progress(false);
                }
            }
        }
        // Auto-clear timer (same as current)
    },
);
```

NOTE: The scan_crash_logs function still accepts a CancellationToken for per-log responsiveness. The `run_cancellable` is an additional outer layer. If scan_crash_logs detects cancellation internally and returns `Ok(ScanResult::cancelled(...))`, that comes through as `Some(Ok(...))` with `cancelled: true`. The bridge-level `None` catches cases where cancellation happens between the bridge and the scan function starting.

**Important constraint:** The DISPATCHER static is an OnceLock that can only be set once per process. Since tests run in the same process, you need to handle this carefully. Options:
1. Use `#[serial_test::serial]` to run tests sequentially (classic-shared-core already has serial_test in dev-dependencies)
2. Design tests that work with a single shared mock dispatcher
3. Use the `get_or_init` fallback behavior

Best approach: Create a MockDispatcher that records dispatch calls using an Arc<Mutex<Vec<...>>>. Set it once with `set_dispatcher` in a test helper. All tests share this dispatcher. Since the OnceLock can only be set once, use a module-level initialization pattern.

Actually, re-read the get_dispatcher() implementation -- it uses `get_or_init` with SlintDispatcher as fallback. For tests, we need to call `set_dispatcher(mock)` BEFORE any test calls a bridge method. But OnceLock::set fails silently if already set.

Better approach for test isolation: Add a `#[cfg(test)]` reset function or use a different pattern for tests. The simplest approach:

Create a `MockDispatcher` that uses `Arc<Mutex<Vec<bool>>>` to track dispatches. Instead of actually calling closures (which might need Slint), just record that dispatch was called and optionally invoke the closure in-place.

```rust
#[cfg(test)]
mod tests {
    use super::*;
    use std::sync::{Arc, Mutex};
    use std::time::Duration;

    /// Mock dispatcher that executes closures immediately (synchronously)
    /// and tracks dispatch count for verification.
    struct MockDispatcher {
        dispatch_count: Arc<Mutex<u32>>,
    }

    impl MockDispatcher {
        fn new() -> (Self, Arc<Mutex<u32>>) {
            let count = Arc::new(Mutex::new(0));
            (Self { dispatch_count: count.clone() }, count)
        }
    }

    impl EventLoopDispatcher for MockDispatcher {
        fn dispatch(&self, f: Box<dyn FnOnce() + Send + 'static>) -> Result<(), BridgeError> {
            *self.dispatch_count.lock().unwrap() += 1;
            f(); // Execute immediately for test verification
            Ok(())
        }
    }

    // Tests that validate the bridge module's types and logic
    // without requiring a Slint event loop.

    #[test]
    fn test_bridge_error_display() {
        let timeout_err = BridgeError::Timeout(Duration::from_secs(5));
        assert!(timeout_err.to_string().contains("5s"));

        let cancelled_err = BridgeError::Cancelled;
        assert!(cancelled_err.to_string().contains("cancelled"));

        let dispatch_err = BridgeError::DispatchFailed("event loop stopped".to_string());
        assert!(dispatch_err.to_string().contains("event loop stopped"));
    }

    #[test]
    fn test_slint_dispatcher_type_exists() {
        // Compile verification -- SlintDispatcher is constructible
        let _ = SlintDispatcher;
    }

    #[test]
    fn test_bridge_error_is_send_sync() {
        fn assert_send_sync<T: Send + Sync>() {}
        assert_send_sync::<BridgeError>();
    }

    #[test]
    fn test_mock_dispatcher_implements_trait() {
        let (mock, count) = MockDispatcher::new();
        let result = mock.dispatch(Box::new(|| {}));
        assert!(result.is_ok());
        assert_eq!(*count.lock().unwrap(), 1);
    }

    #[test]
    fn test_mock_dispatcher_executes_closure() {
        let (mock, _) = MockDispatcher::new();
        let flag = Arc::new(Mutex::new(false));
        let flag_clone = flag.clone();
        let result = mock.dispatch(Box::new(move || {
            *flag_clone.lock().unwrap() = true;
        }));
        assert!(result.is_ok());
        assert!(*flag.lock().unwrap());
    }

    // NOTE: Testing run_with_ui_update, run_with_timeout, and run_cancellable
    // requires the DISPATCHER OnceLock to be set AND a running Tokio runtime.
    // Since the global DISPATCHER can only be set once per process and tests
    // share the process, integration-level tests for the bridge methods
    // should verify at the classic-gui crate level where the full stack is available.
    //
    // The unit tests here validate:
    // - Error types and Display impls
    // - Dispatcher trait contract
    // - Mock dispatcher behavior
    // - Type safety (Send + Sync bounds)
}
```

If there are practical ways to test the async bridge methods themselves (e.g., by using the Tokio test runtime and setting the dispatcher before any test uses it), add those tests. But be realistic about OnceLock limitations. Document what IS tested and what requires integration testing.

Consider adding a `#[tokio::test]` that uses a mock dispatcher via `set_dispatcher` if it can be the FIRST test to run (use test ordering or a Once guard). But if this proves flaky due to test ordering, stick with the type-level and dispatcher-contract tests above.
