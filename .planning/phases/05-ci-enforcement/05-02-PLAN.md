Per D-13: GitHub requires a status check to have completed at least once before it can be registered as required. Per D-11: branch protection is configured manually via GitHub Settings.

**Human verification steps:**

**Step 1: Verify CXX gate CI run (CI-03)**
1. Open the GitHub Actions page for the PR or push that includes the Plan 01 changes
2. Confirm the "CXX Parity Gate" job completed successfully (green check)
3. Confirm "CLI Tests" and "GUI Tests" jobs ran AFTER "CXX Parity Gate" (verify they show dependency)

**Step 2: Verify Python gate remains green (CI-01)**
1. On the same CI run or a recent run, confirm "Python Parity Gates" job in `ci-python-bindings.yml` is green
2. Alternative: run `gh run list --workflow ci-python-bindings.yml --limit 1 --json conclusion` and verify `"conclusion": "success"`

**Step 3: Verify Node gate remains green (CI-02)**
1. On the same CI run or a recent run, confirm "Node Parity Gates" job in `ci-typescript.yml` is green
2. Alternative: run `gh run list --workflow ci-typescript.yml --limit 1 --json conclusion` and verify `"conclusion": "success"`

**Step 4: Configure branch protection (CI-04)**
1. Go to GitHub repository Settings > Branches
2. Click "Edit" on the `main` branch protection rule
3. Under "Require status checks to pass before merging":
   a. Search for "CXX Parity Gate" -- add it as a required check
   b. Search for "Python Parity Gates" -- verify it is listed (add if missing)
   c. Search for "Node Parity Gates" -- verify it is listed (add if missing)
4. Save the branch protection rule

**Step 5: Verify branch protection is active**
1. Open any open PR (or create a test PR)
2. Confirm all three status checks appear in the PR's merge checks section:
   - "CXX Parity Gate"
   - "Python Parity Gates"
   - "Node Parity Gates"
