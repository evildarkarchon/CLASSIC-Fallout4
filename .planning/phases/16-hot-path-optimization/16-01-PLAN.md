<task type="auto">
  <name>Task 3: Analyze Flamegraphs and Document Hot Path Rankings</name>
  <files>.planning/phases/16-hot-path-optimization/16-01-ANALYSIS.md</files>
  <action>
Analyze the collected flamegraphs to identify and rank hot paths by CPU time:

1. **Visual Analysis**: Open flamegraphs in browser and speedscope (https://speedscope.app) to identify wide (time-consuming) stack frames.

2. **Identify Candidates**: Look for:
   - Rust functions consuming >5% of total time
   - PyO3 boundary functions with significant overhead
   - Repeated patterns across all 5 runs (consistent hot paths)
   - Python functions that could be migrated to Rust

3. **Rank Hot Paths**: Create ordered list by cumulative CPU time percentage:
   - Function/method name
   - Approximate % of total time
   - Crate/module location
   - Optimization potential (high/medium/low)

4. **Categorize by Optimization Type**:
   - FFI batching candidates (many small crossings)
   - Cache improvement candidates (low hit rate, check via dump_cache_stats.ps1)
   - SIMD candidates (string/pattern operations)
   - Allocation reduction candidates (string-heavy paths)
   - Algorithm improvement candidates

5. **Document Findings**: Create 16-01-ANALYSIS.md with:
   - Summary of profiling methodology
   - Hot path rankings table (top 5-10)
   - Recommended optimization targets (minimum 3)
   - Proposed optimization techniques for each target
   - Any unexpected findings (e.g., Python hot paths, library overhead)

Run cache stats to correlate with profiling:
