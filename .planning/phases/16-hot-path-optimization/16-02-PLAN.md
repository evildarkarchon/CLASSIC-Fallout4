  <action>
Implement optimizations for the approved hot path targets from 16-01-ANALYSIS.md.

**For each optimization target, follow this pattern:**

1. **Understand the current implementation**: Read the source code for the hot path function.

2. **Apply optimization technique** based on category:

   **FFI Batching** (if PyO3 boundary overhead is hot):
   - Combine multiple small FFI calls into batch operations
   - Return results in single call instead of multiple crossings
   - Example: `process_batch(items: Vec<T>)` instead of loop of `process_single(item)`

   **Cache Optimization** (if cache hit rate is low):
   - Pre-warm caches before hot loops
   - Adjust cache sizing if needed
   - Add strategic cache insertions

   **String Allocation Reduction** (if allocation-heavy):
   - Use `&str` instead of `String` where possible
   - Reuse buffers in hot loops
   - Consider `compact_str` or `smallvec` for small strings/collections

   **SIMD Pattern Matching** (if pattern matching is hot):
   - Use `memchr` crate for SIMD-accelerated search
   - Add to workspace dependencies if not present:
     ```toml
     memchr = "2.7"
     ```

   **Alternative Allocator** (if allocation overhead is significant):
   - Add mimalloc as optional feature:
     ```toml
     [features]
     mimalloc = ["dep:mimalloc"]
     [dependencies]
     mimalloc = { version = "0.1.43", optional = true }
     ```
   - Enable via `#[global_allocator]` with feature flag

   **Algorithm Improvement** (if algorithmic inefficiency):
   - Profile-guided refactoring
   - Early exit conditions
   - Better data structure choices

3. **Maintain API compatibility**: Add new faster APIs alongside existing if needed, mark old as deprecated.

4. **Run quick benchmark** after each optimization to verify improvement:
   ```powershell
   ./scripts/bench/run_benchmarks.ps1 -Mode quick -Filter "[function_name]"
   ```

5. **Ensure tests pass**:
   ```bash
   uv run pytest tests/rust_integration/ -v
   cargo test -p [crate-name]
   ```

**Implementation order**: Follow priority from 16-01-ANALYSIS.md (highest impact first).

**Stop criterion**: If a hot path resists optimization (< 10% improvement after reasonable effort), document why and move to next target.
