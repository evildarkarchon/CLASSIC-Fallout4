**Create `rust/ui-applications/classic-gui/src/markdown.rs`:**

Define `MarkdownBlock` struct with fields matching what Slint will consume:
- `block_type: i32` (0=paragraph, 1=heading, 2=code_block, 3=rule, 4=list_item, 5=blockquote)
- `text: String` (rendered text content, no markdown syntax)
- `heading_level: i32` (1, 2, or 3 for headings)
- `is_bold: bool`
- `is_italic: bool`
- `indent_level: i32` (nesting depth for list items, 0-based)
- `bullet_marker: String` (bullet character for list items)

Define block type constants: `BLOCK_PARAGRAPH=0`, `BLOCK_HEADING=1`, `BLOCK_CODE=2`, `BLOCK_RULE=3`, `BLOCK_LIST_ITEM=4`, `BLOCK_BLOCKQUOTE=5`.

Implement constructor methods on `MarkdownBlock`:
- `heading(level, text)` -- is_bold=true always
- `paragraph(text, bold, italic)`
- `code_block(text)`
- `rule()` -- empty text
- `list_item(depth, text, bold)` -- depth is 1-based from pulldown-cmark's list nesting. Bullet markers per depth: depth 1 = "\u{2022}" (filled bullet), depth 2 = "\u{25E6}" (open circle), depth 3+ = "\u{25AA}" (small square). indent_level = depth - 1 (0-based for Slint padding calculation)
- `blockquote(text, bold)`

Implement `pub fn parse_markdown(source: &str) -> Vec<MarkdownBlock>`:
- Use `pulldown_cmark::Parser::new(source)` with default CommonMark options
- Track state with: `current_text: String`, `in_heading: bool`, `heading_level: i32`, `in_code_block: bool`, `in_strong: bool`, `in_emphasis: bool`, `in_list_item: bool`, `list_depth: i32`, `in_blockquote: bool`
- Handle events following the pattern in RESEARCH.md code example
- For `Event::Text(text)` -- append to current_text
- For `Event::Code(code)` -- append with surrounding backtick characters (inline code stays as text markers since Slint Text cannot do partial background)
- For `Event::SoftBreak` -- append space
- For `Event::HardBreak` -- append newline
- At paragraph end, if current_text is not empty, push paragraph block with accumulated bold/italic state, then clear state
- At heading end, push heading block
- At code block end, push code_block. Trim trailing newline from code block text.
- At list item end, push list_item with current list_depth
- At blockquote end, push blockquote if current_text not empty
- `Event::Rule` pushes a rule block immediately
- Track `list_depth` by incrementing on `Tag::List(_)` start and decrementing on `TagEnd::List(_)` end
- Ignore: HTML events, footnotes, images, table events, metadata events

**Important inline formatting strategy (per user decisions and RESEARCH.md):**
- Block-level formatting flattening: a paragraph containing bold text becomes a bold paragraph
- `in_strong` is set true when `Tag::Strong` starts, stays true for the duration of the block
- `in_emphasis` is set true when `Tag::Emphasis` starts, stays true for the duration of the block
- Reset both at the end of each block (paragraph/list-item/blockquote)

**Create a helper function to push parsed blocks to the Slint model:**

```rust
/// Parse markdown content and push blocks to the Slint report-blocks model
fn update_report_blocks(window: &MainWindow, markdown_content: &str) {
    let blocks = parse_markdown(markdown_content);
    let model = Rc::new(VecModel::default());
    for block in &blocks {
        model.push(MarkdownBlock {
            block_type: block.block_type,
            text: SharedString::from(block.text.as_str()),
            heading_level: block.heading_level,
            is_bold: block.is_bold,
            is_italic: block.is_italic,
            indent_level: block.indent_level,
            bullet_marker: SharedString::from(block.bullet_marker.as_str()),
        });
    }
    window.set_report_blocks(ModelRc::from(model));
}
```

NOTE: `MarkdownBlock` here is the Slint-generated struct from `slint::include_modules!()`. The Rust struct from `classic_gui::MarkdownBlock` is the intermediate representation. They have matching field names but are different types. If there is a naming conflict, rename the import: `use classic_gui::MarkdownBlock as MarkdownBlockData;` and use the Slint-generated `MarkdownBlock` for the Slint model push.
