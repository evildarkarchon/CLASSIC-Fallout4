# Performance Baselines

This directory contains performance baseline data generated by the async pipeline tests.

## Files

- `async_pipeline_baseline_latest.json` - The most recent baseline data (overwritten on each test run)
- `async_pipeline_baseline_YYYYMMDD_HHMMSS.json` - Timestamped baseline data for historical tracking

## Baseline Data Structure

Each JSON file contains the following performance metrics:

```json
{
  "test_date": "2023-XX-XX XX:XX:XX",
  "log_count": 30,
  "total_time": 1.2345,
  "throughput": 24.3,
  "pipeline_efficiency": 0.998,
  "stage_breakdown": {
    "reformat_percent": 46.9,
    "load_percent": 23.9,
    "process_percent": 17.2,
    "write_percent": 11.8
  }
}
```

## Usage

### Running Performance Tests

```bash
# Run the comprehensive baseline test
python -m pytest tests/test_async_pipeline.py::TestAsyncPerformanceBaselines::test_comprehensive_pipeline_baseline -s
```

### Comparing Performance

You can use these baseline files to:
- Track performance improvements over time
- Detect performance regressions
- Compare performance across different system configurations
- Identify which pipeline stages need optimization

## Metrics Explanation

- **total_time**: Total time to process all crash logs (seconds)
- **throughput**: Number of logs processed per second
- **pipeline_efficiency**: Ratio of productive time vs. total time (closer to 1.0 is better)
- **stage_breakdown**: Percentage of time spent in each pipeline stage
  - **reformat_percent**: Time spent reformatting logs
  - **load_percent**: Time spent loading files
  - **process_percent**: Time spent processing content
  - **write_percent**: Time spent writing reports

## Performance Targets

Based on baseline testing:
- **Target throughput**: >40 logs/sec
- **Target efficiency**: >95%
- **Target processing time**: <0.025s per log

## Notes

- Performance data is generated during test runs with realistic but mocked operations
- Actual performance may vary based on system specifications and crash log complexity
- This data is primarily for tracking performance trends and detecting regressions
