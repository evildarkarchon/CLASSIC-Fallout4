# Product Guidelines

## Visual Identity
*   **Aesthetic:** Modern & Minimalist. The GUI should prioritize clean lines, high-contrast typography, and a functional layout that emphasizes clarity and ease of navigation.
*   **Color Palette:** Native Dark Mode support is mandatory. The interface should utilize a dark theme by default, consistent with modern gaming and developer tools.

## Communication & Tone
*   **Prose Style:** Professional & Informative. All user-facing text (error messages, help dialogs, suggestions) must be concise, objective, and provide clear, actionable steps for the user.
*   **Technical Accuracy:** Prioritize technical precision over conversational flair.

## Documentation Standards
*   **In-App Help:** Provide context-sensitive tooltips and clear explanations for all settings and scanning options.
*   **External Documentation:** Maintain comprehensive external documentation (e.g., PDF, Wiki) for complex setup scenarios and advanced troubleshooting.
*   **Code Documentation:**
    *   **Python:** Google-style docstrings are mandatory for all public modules, classes, and methods.
    *   **Rust:** `///` doc comments are mandatory for all public items.

## User Interface Principles
*   **GUI Feedback:** Ensure the interface remains responsive during background operations. Use progress bars and status indicators to provide real-time feedback.
*   **CLI Efficiency:** The command-line interface should be fast and low-overhead, suitable for power users and automated scripts.
