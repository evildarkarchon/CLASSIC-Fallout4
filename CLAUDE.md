## Project Overview

CLASSIC (Crash Log Auto Scanner & Setup Integrity Checker) is a Python application for scanning crash logs generated by Buffout 4 (Fallout 4) and Crash Logger (Skyrim). It provides:
- Crash log analysis with ~250 different checks
- Game/mod file integrity scanning
- Mod file backup/restore functionality
- Script Extender update checking
- Papyrus log monitoring

## Development Commands

### Setup
```bash
# Install Poetry (if not installed)
(Invoke-WebRequest -Uri https://install.python-poetry.org -UseBasicParsing).Content | py -

# Install dependencies
poetry update
```

### Running the Application
```bash
# Run GUI mode
poetry run python CLASSIC_Interface.py

# Run from VS Code: Use F5 or Ctrl+Shift+D, then select "Run GUI"
```

## Architecture Overview

### Entry Points
- **CLASSIC_Interface.py**: GUI application entry point using PySide6
- **CLASSIC_Main.py**: Core initialization and integrity checking logic
- **CLASSIC_ScanLogs.py**: Crash log scanning functionality
- **CLASSIC_ScanGame.py**: Game file scanning and validation

### Core Components

**ClassicLib/** - Main library package containing:
- **Constants.py**: Application constants and enums (YAML stores, DB paths)
- **GlobalRegistry.py**: Singleton pattern for managing global application state
- **Logger.py**: Centralized logging configuration
- **YamlSettingsCache.py**: YAML configuration management with caching
- **Util.py**: Utility functions (e.g., file encoding handling)

**ClassicLib/Interface/** - GUI components:
- **Audio.py**: Sound playback for notifications
- **Papyrus.py/PapyrusDialog.py**: Papyrus log monitoring
- **StyleSheets.py**: Dark mode and UI styling

**ClassicLib/ScanLog/** - Log scanning functionality:
- **DetectMods.py**: Mod detection from crash logs
- **ScanLogInfo.py**: Thread-safe log caching and processing
- **Util.py**: Log parsing utilities

**ClassicLib/ScanGame/** - Game file scanning:
- **CheckCrashgen.py**: Crashgen settings validation
- **CheckXsePlugins.py**: Script Extender plugin validation
- **ScanModInis.py**: Mod INI file scanning

### Data Flow
1. Application initializes via `initialize()` which sets up:
   - YAML settings cache
   - Global registry (game type, VR mode, paths)
   - Logging configuration

2. Main operations:
   - **Crash Log Scanning**: Reads logs from "Crash Logs/" directory, parses with regex, matches against YAML databases
   - **Game Scanning**: Validates game files, checks hashes, scans mod configurations
   - **Results**: Generated as formatted text with emoji indicators (✔️, ❌, ⚠️)

### Key Patterns
- **Singleton Registry**: GlobalRegistry manages application-wide state
- **Thread Safety**: ThreadSafeLogCache for concurrent log processing
- **YAML Configuration**: All settings/rules stored in YAML files under CLASSIC Data/databases/
- **Async Operations**: File operations use asyncio/aiofiles for performance

## Important Notes
- Always use `open_file_with_encoding()` helper for file reading to handle potential unicode issues
- YAML files use ruamel.yaml for preservation of formatting and comments
- Test coverage focuses on thread safety, log parsing, and FormID matching
- GUI runs in separate thread from processing logic
