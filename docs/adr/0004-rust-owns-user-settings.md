# Rust owns User Settings

User Settings comprise every persisted user choice that customizes CLASSIC behavior or remembers frontend state. A dedicated deep Rust module, `classic-user-settings-core`, owns cohesive typed groups, defaults, validation, structured diagnostics, schema evolution, root-relative location rules, and conflict-safe persistence; adapters own presentation and consent, and generic raw YAML operations are not a first-party User Settings interface. The existing nested `CLASSIC Settings.yaml` document evolves semantically losslessly through read-only opens, explicit all-or-nothing updates, and versioned reversible migrations because installed files may contain unknown, legacy, or newer entries; `ClassicConfig` is removed in a breaking cross-language cutover, while `CLASSIC_Info.default_settings` remains only as a generated compatibility mirror.
