# Rust owns Installed YAML Data selection and Local Ignore recovery

`classic-config-core` owns Installed YAML Data selection behind one Rust interface. It independently selects validated Main and game YAML Data from compatible updated and bundled candidates, treats `.prev` only as recovery for a missing canonical cache file, returns immutable snapshots with structured diagnostics, and exposes a side-effect-limited inspection operation so `classic-update-core` does not mirror selection or content-identity rules; explicit test and tooling paths remain a separate mutation-free interface.

Local Ignore YAML Data is user-editable local state, not bundled or update-eligible data: CLASSIC initializes it from the selected Main snapshot when absent and preserves it when present. Malformed Local Ignore YAML Data produces an explicit recovery plan to proceed without ignores for the current operation or reset from the retained defaults after exact-byte conflict checks and a durable verified backup. Reset is one synchronous, non-interruptible Rust critical section: coordination checks cancellation before entry, then lets backup and atomic replacement reach a complete durable outcome before observing cancellation again. It never creates `.prev` update-channel rollback state. A Crash Log Scan Run reports recovery as an expected status and resumes its retained discovery through an opaque, process-local, single-use continuation rather than failing or silently choosing recovery. This extends the Rust-owned operation rule in ADR-0002 so every binding observes the same selection, recovery, cancellation, and diagnostic behavior.
