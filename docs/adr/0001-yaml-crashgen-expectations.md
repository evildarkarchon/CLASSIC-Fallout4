# Use YAML Crashgen Expectations for settings validation

CLASSIC settings validation treats YAML Data-backed Crashgen Expectations as the sole per-Crashgen expectation source. The legacy `checks` key remains accepted and shipped as deprecated inert metadata for YAML compatibility, but scan-time behavior no longer gates on `CheckId` or hardcoded named-check methods. `SettingsValidator` stays as the scanlog adapter over `classic-config-core`'s neutral evaluator, and Disabled Setting Notices remain a separate universal pass included by `scan_all_settings`. This keeps expectations local to YAML Data, avoids two sources of truth, and preserves published YAML compatibility until an explicit schema-breaking cleanup.
