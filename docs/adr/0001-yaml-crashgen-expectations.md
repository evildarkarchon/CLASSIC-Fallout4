# Use YAML Crashgen Expectations for settings validation

CLASSIC settings validation treats YAML Data-backed Crashgen Expectations as the sole per-Crashgen expectation source. The legacy `checks` key remains accepted and shipped as deprecated inert metadata for YAML compatibility, but scan-time behavior no longer gates on `CheckId` or hardcoded named-check methods. `CrashgenSettingsAnalyzer` is the scanlog semantic boundary over `classic-config-core`'s neutral evaluator, and Disabled Setting Notices remain a separate universal result collection. Presentation and placement are owned only by private Autoscan Report Assembly. This keeps expectations local to YAML Data, avoids two sources of truth, and preserves published YAML compatibility until an explicit schema-breaking cleanup.
